## 1차 요구사항 구현
- [x] 유저가 루트 url로 접속시에 게시글 리스트 페이지(http://주소:포트/article/list)가 나온다.
- [x] 리스트 페이지에서는 등록 버튼이 있고 버튼을 누르면 http://주소:포트/article/create 경로로 이동하고 등록 폼이 나온다.
- [x] 게시글 등록을 하면 http://주소:포트/article/create로 POST 요청을 보내어 DB에 해당 내용을 저장한다.
- [x] 게시글 등록이 되면 해당 게시글 리스트 페이지로 리다이렉트 된다. 페이지 URL 은 http://주소:포트/article/list 이다.
- [x] 리스트 페이지에서 해당 게시글을 클릭하면 상세페이지로 이동한다. 해당 경로는 http://주소:포트/article/detail/{id} 가 된다.
- [x] 게시글 상세 페이지에는 id에 맞는 게시글 데이터와 목록 버튼이 있다. 목록 버튼을 누르면 게시글 리스트 페이지로 이동하게 된다.

- (추가 기능이나 구현기능설명이 필요한 경우 서술)

## MVC 패턴
- Model, View, Controller로 응용 프로그램을 나눈 디자인 패턴
- Model : 비즈니스 로직을 수행하는 코드
- View : 모델이 처리한 데이터를 컨트롤러로부터 받아서 시각화
- Controller : 사용자의 요청을 해석하여 처리하고 결과를 반환

## JPA의 장점과 단점
- 장점 : 반복적인 CRUD 작업 자동화, 데이터베이스 종류에 상관없이 일관된 인터페이스 제공, 트랜잭션 관리 등 부가 기능을 쉽게 사용가능, 테스트하기 쉬운 구조 제공
- 단점 : 높은 러닝 커브, N+1 문제 등 

## HTTP GET 요청과 POST 요청의 차이
- GET : 조회
- POST : 생성, 수정
